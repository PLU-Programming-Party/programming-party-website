# DS-01: Project Overview

**Version**: 0.1  
**Date**: 2025-12-15  
**Status**: Draft

## 1. Purpose

**Programming Party Website** - A community hub that explains Programming Party (a weekly gathering of PLU students and alumni), showcases 7 years of creative and silly projects, and invites new members to participate.

The website celebrates the collaborative spirit of building unconventional projects together while serving as a historical archive and recruitment tool.

## 2. Scope

### 2.1 In Scope

- Homepage explaining what Programming Party is and its mission
- Project showcase gallery (7 years of projects)
- Project detail pages with descriptions, links, and repository information
- Integration of existing web-hosted projects (external links)
- Integration of projects with only GitHub repos
- Silly/playful tone and design
- Project metadata (year, team members, status, links)
- Navigation and search/filtering of projects

### 2.2 Out of Scope

- Project hosting/deployment for projects without web presence
- Live demos of every project
- Real-time collaboration features
- User accounts or member authentication
- Event scheduling/calendar for future Programming Party meetings

## 3. Stakeholders

| Role | Name | Responsibility |
|------|------|----------------|
| Product Owner | Programming Party Leadership | Project direction, content approval |
| User Base | PLU students/alumni interested in PP | Feedback, participation |
| Content Manager | TBD | Maintain project database, add new projects |

## 4. Constraints

- Must work with existing project repos (various tech stacks)
- Limited time commitment (volunteer-run project)
- 7 years of projects to catalog/link
- Some projects may have incomplete documentation

## 5. Key Concepts

- **Programming Party**: Weekly creative build sessions for PLU community
- **Project Showcase**: Gallery-style display of all completed projects
- **Silly Projects**: Unconventional, playful software projects
- **Legacy Projects**: Projects from 7 years of history, varying in state
- **Project Status**: Complete, In Progress, Archived, etc.
- **External Integration**: Links to hosted projects and repositories
